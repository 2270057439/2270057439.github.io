第十二部分：多线程

一、进程与线程

1. 进程

Java 为每个运行的程序开启一个进程，即：进程是应用程序的执行示例。进程有独立的内存空间和系统资源，是资源分配的最小单位。

2. 线程

线程是进程中执行运算的最小单位，可完成一个独立的顺序控制流程。一个进程中的所有线程共享进程的资源，除了CPU 时间，也就是说，线程是 CPU 调度和分配的最小单位。

3. 多线程

多线程： 如果在一个进程中同时运行了多个线程，用来完成不同的工作，则称之为“多线程”；多个线程交替占用CPU资源，而非真正的并行执行。

多线程好处

  • 充分利用CPU的资源
  • 简化编程模型
  • 带来良好的用户体验

4. 主线程

  1. main()方法即为主线程入口
  2. 产生其他子线程的线程
  3. 必须最后完成执行，因为它执行各种关闭动作
// 获得主线程对象
Thread t= Thread.currentThread();
// 获取线程名
System.out.println("当前线程是: "+t.getName());
// 设置线程名
t.setName("MyJavaThread");
System.out.println("当前线程名是: "+t.getName());

二、多线程的实现

1. 使用线程的步骤

  1. 定义线程

2. 继承 Thread 类创建线程

  1. 定义 MyThread 类继承 Thread 类
  2. 重写 run() 方法，编写线程执行体
  3. 创建线程对象，调用 start() 方法启动线程

多线程交替执行

Demo02_Thread thread = new Demo02_Thread();
Demo02_Thread thread2 = new Demo02_Thread(); //创建新线程thread2
thread.start();
thread2.start(); //启动thread2

线程应该 start() 而不能直接调用 run()

3. 实现 Runnable 接口创建进程

  1. 实现 Runnable 接口
  2. 创建 Thread 对象，传入 Runnable 对象
  3. 启动线程
class Demo04_Runnable {
    public static void main(String[] args) {
        Runnable runnable = new MyRunnable();
        Thread thread = new Thread(runnable);
        thread.start();
    }
}

class MyRunnable implements Runnable {
    // 实现run()方法
    public void run() {
        for (int i = 1; i < 100; i++) {
            System.out.println(Thread.currentThread().getName() + ":" + i);
        }
    }
}

4. 比较两种创建线程的方式

I. 继承 Thread 类

  • 编写简单，可直接操作线程
  • 适用于单继承

II. 实现 Runnable 接口(推荐使用)

  • 避免单继承局限性
  • 便于共享资源

2. 线程优先级

线程的优先级
线程优先级由1~10表示，1最低，默认优先级为5
优先级高的线程获得CPU资源的概率较大

t1.setPriority(Thread.MAX_PRIORITY);
t2.setPriority(Thread.MIN_PRIORITY);

3. 线程休眠

线程休眠 sleep()
让线程暂时睡眠指定时长，线程进入阻塞状态
睡眠时间过后线程会再进入可运行状态

try {
    Thread.sleep(1000); // 睡眠1秒
} catch (InterruptedException e) {
    e.printStackTrace();
}

4. 线程的强制运行

线程的强制运行 join()
使当前线程暂停执行，等待其他线程结束后再继续执行本线程
需处理InterruptedException异常


注：

  • millis:以毫秒为单位的等待时长
  • nanos:要等待的附加纳秒时长
try {
    // 阻塞主线程，子线程强制执行
    temp.join();
} catch (InterruptedException e) {
    e.printStackTrace();
}

5. 线程的礼让

线程的礼让 yield()
暂停当前线程，允许其他具有相同优先级的线程获得运行机会
该线程处于就绪状态，不转为阻塞状态
public static void yield()

注： 只是提供一种可能，但是不能保证一定会实现礼让

五、线程同步

问题： 多线程实现网络购票，用户提交购票信息后
第一步：网站修改网站车票数据
第二步：显示出票反馈信息给用户

1. 线程不安全的抢票代码

public class Test01_unsafty {
    public static void main(String[] args) {
        // 车站售票点
        Site_unsafty site = new Site_unsafty();
        // 每个线程就是一个抢票的人
        Thread person1 = new Thread(site, "桃跑跑");
        Thread person2 = new Thread(site, "抢票代理");
        Thread person3 = new Thread(site, "黄牛党");
        System.out.println("********开始抢票********");
        person1.start();
        person2.start();
        person3.start();
    }
}

class Site_unsafty implements Runnable {
    private int count = 10; // 记录剩余票数
    private int num = 0; // 记录买到第几张票

    public void run() {
        while (true) {
            // 没有余票时，跳出循环
            if (count <= 0) {
                break;
            }
            // 第一步：修改数据
            num++;
            count--;
            try {
                Thread.sleep(500); // 模拟网络延时
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            // 第二步：显示信息
            System.out.println(Thread.currentThread().getName() + "抢到第" + num
                    + "张票，剩余" + count + "张票！");
        }
    }
}

线程同步问题： 多个线程操作同一共享资源时，将引发数据不安全问题。

2. 使用同步代码块解决同步问题

语法：

synchronized(锁对象){
    // 同步代码块
}

锁的含义：通过锁将一段代码锁定。锁是一个对象，进入锁定的同步代码块，线程要首先持有该锁，只有持有锁的线程可以执行该代码，执行完毕，释放锁；然后，锁可以被其他线程拿到。锁可以是任意对象。

3. 使用同步方法解决同步问题

语法：

public synchronized void buy() {
    // 同步代码块
}

同步方法： 同步方法和同步代码块的作用和内涵是一致的。同步方法即是用当前对象 this 锁定的同步代码块。

4. 线程安全的类型

  • ArrayList类的add()方法为非同步方法
  • 当多个线程向同一个ArrayList对象添加数据时，可能出现数据不一致问题
  • 为达到安全性和效率的平衡，可以根据实际场景来选择合适的类型*
