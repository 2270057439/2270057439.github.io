第十一部分 JDBC

一、JDBC 的工作原理

1. 什么是 JDBC？

JDBC(Java Database Connection)： Java 数据库连接，提供连接各种常用数据库的能力。

2. JDBC 原理

技术 提供者 作用 内容
JDBC API Sun公司 编程接口 供程序员调用的接口与类， 集成在java.sql和javax.sql包中， 如：DriverManager类、Connection接口、 Statement接口、ResultSet接口
DriverManager Sun公司 管理各种不同的JDBC驱动 Java内部类 DriverManager
JDBC 驱动 数据库厂商 负责连接各种不同的数据库 驱动jar包

3. 模板代码

JDBC API主要功能：

  graph LR
  与数据库建立连接-->执行SQL语句
  执行SQL语句-->处理结果

DriverManager： 依据数据库的不同，管理JDBC驱动
Connection： 负责连接数据库并担任传送数据的任务
Statement： 由 Connection 产生、负责执行SQL语句
ResultSet： 负责保存Statement执行后所产生的查询结果

代码模板：

public static void main(String[] args) {
    Connection     conn = null;
    Statement     stmt = null;
    ResultSet    rs    = null;
    try {
        // 1. 加载 mysql jdbc 驱动
        Class.forName("com.mysql.jdbc.Driver");
        // 2. 获取连接对象
        conn = DriverManager.getConnection("jdbc:mysql://localhost:3306/pet_db", "root", "123");
        // 3. 创建语句对象
        stmt = conn.createStatement();
        // 4. 执行语句返回结果
        rs = stmt.executeQuery("select * from dog");
        // 处理查询结果
        while(rs.next()){
            int id = rs.getInt(1);
            String name = rs.getString(2);
            int    health = rs.getInt(3);
            int love = rs.getInt(4);
            String strain = rs.getString(5);
            System.out.println(id + "\t" + name + "\t" + health + "\t" + love + "\t" + strain);
        }            
    } catch (ClassNotFoundException e) {
        e.printStackTrace();
    } catch (SQLException e) {
        e.printStackTrace();
    } finally{
        // 5. 释放对象
        try {
            if(rs != null){
                rs.close();
            }
            if(stmt != null){
                stmt.close();
            }
            if(conn != null){
                conn.close();
            }
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}

JDBC 的 5 个步骤：

  1. 加载驱动
  2. 获得连接
  3. 创建语句对象
  4. 执行语句
  5. 释放资源

一句话就是：加连语执释(贾琏欲执事)
